Strip aspirational v1 features; honest privacy notice; GitHub-issues-only contact

Public-facing docs and the wire schema described several things that are
not actually deployed (Turnstile captcha, BYOK custom-view textbox,
moderation-flag endpoint, daily-rotating IP salt with 7-day expiry, a
v1 roadmap). A reader hitting any of those endpoints, posting bodies,
or relying on the salt-rotation privacy guarantee would get behavior
that didn't match the docs. This commit makes the public surface
match what is actually live.

AGENTS.md
  - §5: drop "(planned for v1)" parenthetical on the genesis-anchoring
    bullet; the framing stays honest about what's not guaranteed.
  - §6 Write table: rename to "Write (rate-limited per IP)"; remove
    turnstile_token from request bodies; remove the unwired
    POST /entries/:id/flag and POST /views (BYOK) rows; add the now-
    deployed POST /bodies + GET /e/:id rows in the right place.
  - §6 Conventions: drop the Turnstile bullet; document the actual
    per-IP rate limit defaults and that they live in rate_limit_config.
  - §8 CORS table: drop the POST /views row.
  - §9 Fork section: drop "what custom-view prompts you suggest".
  - §11 Operator: rewrite as GitHub-issues-only contact, link to the
    Security Advisories surface for security disclosures.
  - §12: replace "v1 will likely add ..." with a one-paragraph
    "this document tracks deployed behavior" note.

privacy.md (rewritten end-to-end)
  - ip_hash now described accurately: fixed internal salt, used for
    rate-limiting and abuse detection. Drops the false claim about

  - captcha_id bullet removed entirely (no Turnstile deployed).
  - LLM section rewritten for the deployed Doc View: the page's post
    bodies (already public) plus our prompt go to OpenAI; no user

  - "What we publish" / "What we don't share" lines drop captcha_id.
  - Erasure flow: email → open a GitHub issue with the templated title.
  - Updates section: drops the /p/about reference (no such page);
    points at GitHub releases.

contact.md (rewritten)

spec.json (machine-readable schema served at /api/spec.json)
  - POST /entries: drop turnstile_token from request properties.
  - POST /pages: drop turnstile_token + default_view_prompt from
    request properties; we don't surface either today.

entries/route.ts: drop the "Phase A" comments + the dead
